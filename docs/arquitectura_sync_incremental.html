Arquitectura de Sincronización Incremental

Documento de referencia para entender cómo la app ahorra lecturas de Firestore, cómo se sincroniza la base local del dispositivo y qué debes actualizar en Firebase cuando agregas o editas categorías y negocios.

Resumen

Qué resuelve esta arquitectura

  • La app ya no usa listeners en tiempo real sobre colecciones grandes.
  • Primero carga datos desde caché local del dispositivo.
  • Después revisa un documento pequeño en app_meta/sync.
  • Si detecta cambios, solo descarga los documentos con actualizado > último sync local.
  • La barra de búsqueda usa la copia local sincronizada en memoria.

Guía operativa

Si agregas un negocio nuevo

  1. Crea el documento en negocios.
  2. Llena todos sus campos, incluyendo creado y actualizado.
  3. Pon activo = true.
  4. Actualiza app_meta/sync.negociosUpdatedAt con el timestamp actual.

Si editas un negocio existente

  1. Modifica los campos necesarios.
  2. Actualiza actualizado del negocio.
  3. Actualiza app_meta/sync.negociosUpdatedAt.

Si quieres ocultar un negocio

  1. No lo borres físicamente si quieres un sync limpio.
  2. Cambia activo = false.
  3. Actualiza actualizado.
  4. Actualiza app_meta/sync.negociosUpdatedAt.

Si agregas o editas una categoría

  1. Asegúrate de que la categoría tenga actualizado.
  2. Si la cambias, actualiza ese timestamp.
  3. Actualiza app_meta/sync.categoriasUpdatedAt.

Qué campos debes tocar siempre

Cambio Documento modificado Documento global
Agregar negocio negocios/{id}.creado y negocios/{id}.actualizado app_meta/sync.negociosUpdatedAt
Editar negocio negocios/{id}.actualizado app_meta/sync.negociosUpdatedAt
Ocultar negocio negocios/{id}.activo = false y actualizado app_meta/sync.negociosUpdatedAt
Agregar o editar categoría categorias/{id}.actualizado app_meta/sync.categoriasUpdatedAt

Ventajas de esta estrategia

  • Reduce mucho las lecturas comparado con usar snapshots() en colecciones grandes.
  • La app sigue funcionando offline si esos datos ya fueron consultados antes.
  • La búsqueda no abre consultas nuevas mientras escribes.
  • Si cambias un solo negocio, ya no necesita volver a bajar todos los negocios.

Importante

  • Usa siempre tipo Timestamp real en Firestore, no texto.
  • Conviene desactivar documentos con activo: false en vez de borrarlos de inmediato.
  • Para que el incremental funcione bien, debes ser consistente con el campo actualizado.
